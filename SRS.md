# Software Requirements Specification (SRS)

## 1. บทนำ

### 1.1 วัตถุประสงค์
ระบบ backend สำหรับจัดการปัญหาน้ำประปาไม่สะอาด มีการเก็บข้อมูลรายงานปัญหา อัปเดตสถานะ และดึงข้อมูลรายงานเพื่อใช้งานกับ frontend หรือระบบบริหาร.

### 1.2 ขอบเขตระบบ
ระบบรองรับการจัดการข้อมูลปัญหาน้ำประปา โดยใช้ API สำหรับ:
- สร้างรายงานปัญหาใหม่
- ดูข้อมูลรายงานทั้งหมด
- ดูข้อมูลรายงานตาม ID
- แก้ไขรายละเอียดและสถานะของรายงาน
- ลบรายงาน

### 1.3 คำนิยาม
- Issue: รายงานปัญหาน้ำประปาไม่สะอาด
- Backend: ส่วนบริการ API และฐานข้อมูล
- API Client: frontend หรือระบบภายนอกที่เรียกใช้งาน API

## 2. ความต้องการทั่วไป

### 2.1 ความต้องการฟังก์ชัน
- ผู้ใช้งานสามารถเพิ่มรายงานปัญหาได้
- ผู้ใช้งานสามารถดูรายงานทั้งหมดได้
- ผู้ใช้งานสามารถดูรายละเอียดรายงานตาม ID ได้
- ผู้ใช้งานสามารถแก้ไขรายงานได้
- ผู้ใช้งานสามารถลบรายงานได้

### 2.2 ความต้องการข้อมูล
ข้อมูลรายงานต้องประกอบด้วย:
- `title` (หัวข้อปัญหา)
- `description` (คำอธิบาย)
- `location` (ตำแหน่ง)
- `reported_by` (ผู้รายงาน)
- `status` (สถานะ เช่น reported, investigated, resolved)
- `created_at` (วันที่สร้าง)
- `updated_at` (วันที่แก้ไขล่าสุด)

### 2.3 ข้อจำกัด
- ใช้ฐานข้อมูล SQLite สำหรับต้นแบบ
- ไม่มีระบบผู้ใช้งานหรือสิทธิ์การเข้าถึง
- API เป็น REST แบบพื้นฐาน

## 3. ความต้องการเชิงไม่ใช่ฟังก์ชัน

### 3.1 ประสิทธิภาพ
- รองรับการดึงข้อมูลจำนวนมากได้อย่างรวดเร็ว
- จำกัดผลลัพธ์ `GET /issues` ได้ด้วย `skip` และ `limit`

### 3.2 ความน่าเชื่อถือ
- ต้องมีการจัดการข้อผิดพลาดเมื่อไม่พบข้อมูล
- ต้องคืนสถานะ HTTP ที่เหมาะสม (404, 201, 200)

### 3.3 ความสามารถในการบำรุงรักษา
- โค้ดต้องมีการแยกชั้น model, schema, crud, main
- เอกสาร API ต้องอ่านง่ายและชัดเจน

## 4. API

### 4.1 `GET /`
- คำอธิบาย: ตรวจสอบสถานะระบบ
- ผลลัพธ์: message จากระบบ

### 4.2 `GET /issues`
- คำอธิบาย: ดึงรายงานทั้งหมด
- พารามิเตอร์: `skip`, `limit`

### 4.3 `GET /issues/{issue_id}`
- คำอธิบาย: ดึงรายงานตาม ID

### 4.4 `POST /issues`
- คำอธิบาย: สร้างรายงานใหม่
- JSON ตัวอย่าง:
  - title
  - description
  - location
  - reported_by
  - status

### 4.5 `PUT /issues/{issue_id}`
- คำอธิบาย: อัปเดตรายงาน
- รองรับการแก้ไขบางฟิลด์

### 4.6 `DELETE /issues/{issue_id}`
- คำอธิบาย: ลบรายงาน

## 5. ความต้องการสภาพแวดล้อม

- Python 3.11+ หรือ 3.13
- FastAPI
- SQLAlchemy
- Uvicorn
- SQLite

## 6. ข้อสมมติฐาน

- ระบบจะเชื่อมต่อกับฐานข้อมูลโลคัลเดียว
- ไม่มีการจัดการการอนุญาตหรือผู้ใช้ในเฟสแรก
- ระบบรองรับเฉพาะ HTTP โดยตรง
